# 记账Web应用 - 测试计划

## 1. 测试概述

### 1.1 测试目标
验证记账Web应用的功能完整性、数据可靠性、用户体验和兼容性，确保产品满足需求文档中的所有验收标准。

### 1.2 测试范围
| 测试类型 | 覆盖内容 |
|---------|---------|
| 功能测试 | 记账条目管理、标签管理、统计报表、数据持久化 |
| UI测试 | 响应式布局、交互反馈、暗色模式(可选) |
| 兼容性测试 | 多浏览器、多设备尺寸 |
| 数据测试 | 数据完整性、导入导出功能 |

### 1.3 测试环境
- **浏览器**: Chrome 80+、Firefox 75+、Safari 13+、Edge 80+
- **设备**: 桌面显示器(1920x1080)、平板(768x1024)、手机(375x667)
- **操作系统**: macOS、Windows、iOS、Android

## 2. 验收标准核对表

| 编号 | 验收标准 | 测试方法 | 优先级 |
|-----|---------|---------|-------|
| AC-01 | 用户可成功添加、编辑、删除记账条目 | 功能测试 | P0 |
| AC-02 | 用户可创建、编辑、删除标签 | 功能测试 | P0 |
| AC-03 | 记账时可选择标签 | 功能测试 | P0 |

## 3. 测试策略

### 3.1 功能测试顺序
1. **数据持久化** - 先验证数据存储基础功能
2. **标签管理** - 标签是记账的基础依赖
3. **记账条目管理** - 核心功能
4. **统计报表** - 依赖记账数据

### 3.2 测试数据准备
- 预置测试标签：餐饮、交通、购物、娱乐、工资
- 预置测试记账条目：覆盖不同类型、金额、日期

### 3.3 缺陷等级定义
| 等级 | 定义 | 示例 |
|-----|-----|-----|
| P0-阻塞 | 核心功能无法使用 | 无法添加记账条目 |
| P1-严重 | 主要功能异常 | 统计数据计算错误 |
| P2-一般 | 次要功能缺陷 | 标签颜色显示异常 |
| P3-建议 | 体验优化建议 | 交互流畅度改进 |
